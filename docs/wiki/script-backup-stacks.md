# backup-stacks.sh

> Restic backup script that pauses labelled Docker containers across all running LXCs, backs up `/opt/appdata`, then resumes containers — even on failure.

## Overview

`scripts/host/backup-stacks.sh` runs on the Proxmox host and coordinates a consistent backup of all application data. It uses `pct exec` to communicate with each running LXC, pausing containers that hold critical state before Restic runs. A `trap cleanup EXIT` guarantees containers are always unpaused, regardless of how the script exits.

## Usage

```bash
./scripts/host/backup-stacks.sh
# or via menu:
./host.sh → Backup Stacks (Restic)
```

## Configuration

Create `scripts/host/.env` on the Proxmox host (`chmod 600`, **never commit**):

```bash
RESTIC_REPOSITORY=s3:https://s3.example.com/mybucket
# or: /mnt/backup-drive/restic-repo
RESTIC_PASSWORD=your-strong-restic-password
```

The script fails immediately with a clear error if either variable is missing.

## Backup Process

1. **Find running LXCs** — `pct list` filtered to `running` state
2. **Pause** — for each LXC with Docker: `docker pause` all containers with `label=com.homelab.backup.pause=true`
3. **Backup** — `restic backup /opt/appdata --cleanup-cache`
4. **Resume** — `docker unpause` all previously paused containers (via `trap`)

The pause is intentionally brief — only for the duration of the Restic snapshot, which for a typical homelab takes seconds to minutes.

## The `com.homelab.backup.pause=true` Label

Any container that must be paused carries this label. It is included in the boilerplate template generated by [lib-stack.sh](lib-stack.md):

```yaml
labels:
  - "com.homelab.backup.pause=true"
```

Containers that should be paused: databases (PostgreSQL, Redis), media servers (Jellyfin), download clients (qBittorrent), Paperless. Stateless services (Watchtower, GoAccess) do not need this label.
