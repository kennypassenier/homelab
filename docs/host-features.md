

# Tier 2: HOST (Proxmox Bare-Metal Daemon)

The HOST is a native `systemd` daemon running directly on the Proxmox bare-metal hypervisor, replacing `host.sh` and host-side bash scripts ([5], [6]).

## 1. Premium UI/UX (Ratatui)
- [ ] **Hyper-Modern Interface:** The host daemon must feature a `ratatui` TUI that matches the premium aesthetic of the CLIENT ecosystem.
- [ ] **Styling & Feedback:** Uses a consistent color palette (Cyan/Magenta accents, dark grey backgrounds). All Layout blocks must use rounded borders (`BorderType::Rounded`). 
- [ ] **Dynamic States:** The Dashboard and Backup tabs must highlight when active. Long-running tasks (like Restic backups or LXC cloning) must display animated spinners. Confirmation modals (like rebooting an LXC or wiping a stack) must render as floating, centered pop-ups with a 3D shadow effect.

## 2. Infrastructure Provisioning
- [ ] **Proxmox API Client:** Uses `reqwest` to interact with the Proxmox API. It clones Debian templates to create new LXCs and injects the MAC addresses generated by the CLIENT (replacing the old `bootstrap-lxc.sh` flow).
- [ ] **Post-Provisioning Exec Hook:** Immediately after LXC creation, the HOST daemon uses the Proxmox Exec API to atomically run `apt-get update && apt-get upgrade`, install Docker ([7]), and install `unattended-upgrades` for automatic OS security patching ([8], [9]). If the `apt` command fails, the HOST daemon implements a Fail-Safe and deletes the broken LXC immediately.

## 3. Atomic Hardware Passthrough
- [ ] **GPU & TUN Parsing:** Replaces `enable-gpu.sh` and `enable-tun.sh` ([10], [11]). The daemon reads `/etc/pve/lxc/<vmid>.conf` to configure hardware acceleration (e.g., for Jellyfin [12]) or VPN tunnels (e.g., for Gluetun [13]).
- [ ] **Atomic Rename:** To prevent file corruption, it writes the new `lxc.cgroup2.devices.allow` rules to a temporary file, then performs an atomic `rename` operation to overwrite the config.

## 4. Backup Orchestration & Stack Recovery
- [ ] **API-Driven Restic Backups:** Replaces `backup-stacks.sh` ([14]). Instead of manually pausing containers, the HOST sends an HTTP POST `/api/backup/pause` to the LXC Daemon.
- [ ] **Rust Drop Guards:** After Restic backs up `/opt/appdata` ([15]), a Rust Drop Guard guarantees that the `/api/backup/resume` signal is sent to the LXC, even if the Restic process panics or crashes.
- [ ] **Stack Reset Functionality:** Replaces `reset-stack.sh`. Safely wipes corrupted Docker state inside the LXC via the Exec API and atomically deletes the `/opt/appdata/<STACK>` data from the host, while preserving the LXC container itself.

## 5. CI/CD Self-Updater
- [ ] When changes are pushed to the `host-daemon/` folder, GitHub Actions compiles the Rust binary and publishes it to GitHub Releases.
- [ ] The HOST daemon periodically checks the GitHub Releases API, downloads the new binary to `/tmp/`, atomically overwrites its current executable, and restarts itself via `systemctl`.
