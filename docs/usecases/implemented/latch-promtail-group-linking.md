# Use Case: Latch Group Linking for Promtail .env Files

**Tier:** CLIENT + LXC
**Status:** Planned

## Goal

Use the latch `# latch:group=<name>` pragma to link all identical promtail `.env` files under one shared encrypted blob, eliminating redundant secret copies and making Loki IP changes a single-edit operation.

## Why Useful

Every stack ships a promtail sidecar. All promtail containers point to the same Loki instance, so their `.env` files are byte-for-byte identical. Without grouping, each stack's secret is a separate encrypted blob. Changing the Loki IP requires updating six blobs independently. The latch group model encrypts the shared content once (`group.promtail_config.enc`) and fans it out to all members on pull — a single source of truth for a shared credential.

## Behavior

- The git-tracked `stacks/<stack>/promtail/.env` contains only the group pragma (subscribe-intent member):
  ```dotenv
  # latch:group=promtail_config
  ```
- At runtime, `pre-sync.sh` writes the actual `.env` to `/appdata/<stack>/promtail/.env` via `latch pull` or the stack-specific sync hook. The pragma file is not used at runtime — it is a latch metadata hint only.
- When `latch push` is run, discovery reads the pragma, groups all members under `promtail_config`, encrypts the shared content once, and stores it as `group.promtail_config.enc`.
- When `latch pull` is run, the group blob is decrypted and fanned out to all member paths.
- `latch group list` / `latch group show promtail_config` show current membership.
- `latch status` reports the group as a single sync unit.

## Affected Files

- `scripts/client/lib/lib-stack.sh` — `generate_promtail()` creates the pragma `.env` in the git-tracked stack dir
- `stacks/*/promtail/.env` — each is a subscribe-intent member with only the pragma line (runtime `.env` is generated by pre-sync, not tracked in git)

## Implementation Notes

- Pragma must be the **first line** of the `.env` file.
- Group name validated to `[a-zA-Z0-9_-]+` — `promtail_config` is valid.
- Subscribe-intent members (pragma-only, no key/value pairs) pull from the existing remote group blob; they do not define content.
- The Latch sync step in `pre-sync.sh` writes the runtime `.env` to `/appdata/` and is unaffected by this change.
